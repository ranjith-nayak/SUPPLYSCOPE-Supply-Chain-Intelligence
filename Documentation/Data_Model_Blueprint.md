# Data Model Blueprint - SUPPLYSCOPE

## Overview

The SUPPLYSCOPE data model is designed using a star schema architecture to support supply chain analytics, operational intelligence, logistics monitoring, inventory analysis, supplier evaluation, and demand forecasting.

The model centers around a primary fact table containing transactional order data and multiple dimension tables that provide descriptive business context.

---

# Data Model Architecture

The model follows a star schema structure.

```text
                 Dim_Product
                       |
                       |
Dim_Supplier --- Fact_Orders --- Dim_Warehouse
                       |
                       |
                 Dim_Shipment
                       |
                       |
                  Dim_Date
                       |
                       |
                  Dim_Region
```

Fact_Orders serves as the central table and connects all business dimensions.

---

# Fact Table

## Fact_Orders

Purpose:

Stores transactional order-level data generated by customer purchases and fulfillment activities.

### Primary Key

* Order_ID

### Columns

| Column Name        | Description                 |
| ------------------ | --------------------------- |
| Order_ID           | Unique order identifier     |
| Order_Date         | Date of order placement     |
| Product_ID         | Product reference           |
| Supplier_ID        | Supplier reference          |
| Warehouse_ID       | Warehouse reference         |
| Shipment_ID        | Shipment reference          |
| Region_ID          | Customer region reference   |
| Quantity           | Units ordered               |
| Revenue            | Revenue generated           |
| Fulfillment_Status | Fulfilled, Delayed, Pending |

# Business Logic Flow

The supply chain follows the process below:

Supplier
↓
Warehouse
↓
Inventory
↓
Order Fulfillment
↓
Shipment
↓
Customer

Every stage generates operational data that is captured and analyzed within SUPPLYSCOPE.
